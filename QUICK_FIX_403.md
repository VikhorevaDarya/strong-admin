# ⚡ Быстрое исправление ошибки 403

## Проблема
```
ClientResponseError 403: Only admins can perform this action.
```

## Решение за 2 минуты

### 1️⃣ Откройте PocketBase админку
```
http://127.0.0.1:8090/_/
```

### 2️⃣ Исправьте коллекцию "products"

1. Кликните на коллекцию **products**
2. Нажмите на вкладку **API Rules** (правила API)
3. Вы увидите 5 полей:
   - List/Search rule
   - View rule
   - Create rule
   - Update rule
   - Delete rule

4. В **КАЖДОЕ** поле вставьте:
   ```
   @request.auth.id != ""
   ```

5. Нажмите **Save** (Сохранить)

### 3️⃣ Исправьте коллекцию "warehouses"

1. Кликните на коллекцию **warehouses**
2. Вкладка **API Rules**
3. В **КАЖДОЕ** из 5 полей вставьте:
   ```
   @request.auth.id != ""
   ```

4. Нажмите **Save**

### 4️⃣ Перезагрузите админку

1. Откройте консоль браузера (F12)
2. Выполните:
   ```javascript
   localStorage.clear();
   location.reload();
   ```

3. Войдите заново
4. Попробуйте создать товар

---

## ✅ Готово!

Теперь вы сможете создавать, редактировать и удалять товары и склады.

---

## 🔍 Что означает это правило?

```
@request.auth.id != ""
```

- `@request.auth.id` - ID авторизованного пользователя
- `!= ""` - НЕ пустой (то есть пользователь ДОЛЖЕН быть авторизован)

Это правило разрешает доступ **любому авторизованному пользователю**.

---

## ⚠️ Если не помогло

1. Убедитесь что PocketBase запущен: `./pocketbase serve`
2. Проверьте что вы вошли в систему (есть токен в localStorage)
3. Проверьте консоль браузера на наличие других ошибок
4. Убедитесь что в API Rules написано именно `!=` а не `=`
